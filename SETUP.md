# Как запустить sber26-bot

Эта инструкция для **не-программиста**. Если что-то непонятно или команда
не работает — скинь скрин / текст ошибки, разберёмся.

Всё делается **один раз**. Дальше бот просто работает, а каждый push в
`main` на GitHub автоматически передеплоивается.

---

## Шаг 1: Создать Telegram-бота

1. Открой в Telegram чат с **@BotFather**.
2. Напиши `/newbot`.
3. Он спросит имя бота — пиши что угодно (это что люди видят в чате).
4. Потом спросит username — должен кончаться на `bot`, например
   `sber26_acc_bot`. Если занят — придумай другой.
5. Он пришлёт сообщение с **токеном** — выглядит примерно так:
   ```
   1234567890:ABCdefGhIJKlmnoPQRsTUVwxyz
   ```
   **Скопируй и сохрани.** Он понадобится на Шаге 4.
6. Пока у BotFather — отключи privacy mode (это нужно чтобы бот видел
   сообщения в группе, а не только команды со слэшем):
   - напиши `/mybots` → выбери бота → **Bot Settings** → **Group Privacy**
     → **Turn off**.
7. Там же: **Allow Groups?** → **Enable** (если вдруг выключено).

---

## Шаг 2: Получить API-ключ Anthropic

1. Зайди на https://console.anthropic.com.
2. Залогинься. **ВАЖНО:** аккаунт на console.anthropic.com —
   ОТДЕЛЬНЫЙ от Claude Max-подписки. Если учётки нет — зарегай (ту же
   почту можно, пароль они попросят свой).
3. **Top up balance** на $20 — хватит на пару месяцев с запасом
   (Settings → Billing → Add funds).
4. Создай API-ключ: **Settings → API Keys → Create Key**.
5. Дай название типа `sber26-bot-prod`, скопируй ключ. Он начинается с
   `sk-ant-…`. **Больше он не покажется** — сохрани сразу.

---

## Шаг 3: Развернуть на Railway

1. Зайди на https://railway.app и залогинься через GitHub.
2. **New Project** → **Deploy from GitHub repo**.
3. Если Railway впервые видит твой GitHub — нажми **Configure GitHub App**
   → выбери `AlKazBot` (либо дай доступ ко всем репам, как удобнее).
4. Выбери `daanameporia-ux/AlKazBot`. Railway создаст сервис и начнёт
   сборку — первый билд минут 3-5, не трогай.
5. Пока билдится — добавь Postgres:
   - В том же проекте жми **+ New** → **Database** → **Add PostgreSQL**.
   - Railway сам создаст базу и **автоматически подставит** переменную
     `DATABASE_URL` в сервис бота — руками ничего делать не надо.
6. Теперь переменные окружения. Открой сервис бота (не Postgres!) →
   вкладка **Variables** → добавь по одной:

   | Имя | Значение |
   |---|---|
   | `TELEGRAM_BOT_TOKEN` | токен из Шага 1 |
   | `ANTHROPIC_API_KEY` | ключ из Шага 2 |
   | `ANTHROPIC_MODEL` | `claude-sonnet-4-6` |
   | `ALLOWED_TG_USER_IDS` | `6885525649,7220305943` (Казах + Арбуз) |
   | `OWNER_TG_USER_ID` | `6885525649` (Казах) |
   | `APP_ENV` | `prod` |
   | `LOG_LEVEL` | `INFO` |
   | `HYBRID_LISTEN_MODE` | `true` |
   | `ENABLE_PRANKS` | `false` |
   | `MAIN_CHAT_ID` | **пока оставь `0`**, заполним после Шага 6 |

   `DATABASE_URL` Railway уже добавил автоматически из Postgres-сервиса
   — проверь что видно в списке, и не трогай.

7. Railway передеплоит с новыми переменными. Смотри вкладку **Deploy Logs**:
   должно закончиться строкой типа `bot_starting app_env=prod …`.

   Если там ошибка — скинь скрин логов.

## Шаг 4: Telegram user ID — уже зашиты

Твой ID (Казах) `6885525649` и ID Арбуза `7220305943` уже прописаны в
Шаге 3.6 и засеяны в базу (миграция `c1a00002seed`). Никаких действий
— пропускаем.

Если надо добавить помощника в whitelist — отредактируй
`ALLOWED_TG_USER_IDS` в Railway Variables, запятая между ID.

## Шаг 5: Добавить бота в групповой чат

1. В Telegram открой свой рабочий групповой чат.
2. Настройки чата → **Add members** → найди своего бота по username
   (`@sber26_acc_bot` или как назвал) → добавь.
3. Сделай бота **администратором**:
   - Настройки чата → **Administrators** → **Add Administrator** →
     выбери бота → включи права:
     - Delete messages ✓
     - Pin messages ✓
     - Change chat info ✓ (опционально, для будущих приколов)

---

## Шаг 6: Прописать MAIN_CHAT_ID

1. В групповом чате напиши боту: `/chatid`.
2. Бот ответит сообщением вида:
   ```
   chat_id = `-1001234567890`
   your user_id = `123456789`
   ```
3. Скопируй `chat_id` (с минусом!).
4. Railway → сервис → **Variables** → замени `MAIN_CHAT_ID` на это
   значение. Railway передеплоится.

---

## Шаг 7: Проверка

В групповом чате напиши `/start`. Бот должен ответить приветствием
("Здарова, команда. Я тут новенький …").

Если не отвечает:
- Проверь что ты в `ALLOWED_TG_USER_IDS`.
- Проверь Deploy Logs на Railway — нет ли там ошибок.
- Privacy mode у BotFather выключен? (Шаг 1.6).
- Напиши мне в другой чат / сюда — скинь логи.

---

## Частые проблемы

**Бот молчит.** 99% случаев — не выключен Privacy mode (Шаг 1.6) или
юзер не в `ALLOWED_TG_USER_IDS`.

**Deploy failed / зависает.** Открой **Build Logs** — обычно там
сообщение вида `Could not find X`. Скинь скрин.

**Бот ответил один раз и перестал.** Скорее всего Railway прибил
сервис за неактивность или превысил free-tier. Проверь **Usage** в
проекте — если исчерпан лимит, надо оплатить Pro план ($5/мес).

**Миграции не применились.** `Deploy Logs` должны показать
`alembic ... upgrade head ... Running upgrade -> <hash>`. Если не
видно — сервис не успевает стартануть; проверь там же ошибки.

## Как научить бота чему-то новому

В чате тегни бота и напиши `запомни:` — например:

```
@sber26_acc_bot запомни: поставщик Миша — всегда 22-28к за кабинет,
если цена сильно отличается, переспрашивай
```

Бот запишет это как правило и будет учитывать при следующих сообщениях.

> ⚠️ Команды обучаемости (`/knowledge`, `запомни: ...`) реально работают
> с **Этапа 1**. Сейчас бот на Этапе 0 — отвечает только на /start, /help
> и /chatid, и молча логирует всё остальное.

## Как посмотреть что бот знает

`/knowledge` — покажет список.

## Как удалить что-то из памяти

`/knowledge forget <номер>` — номер берёшь из списка.
